## 核心模块

### 1. TykeFramework

框架核心类，管理整个应用的生命周期。

**职责：**
- 初始化日志系统
- 初始化线程池
- 启动 IPC 服务器
- 管理控制器注册

**设计模式：** 单例模式

### 2. IpcServer

IPC 服务器，处理进程间通信。

**职责：**
- 接受客户端连接
- 数据加密/解密
- 消息分发

**设计模式：** PIMPL 模式（隐藏平台实现）

### 3. ThreadPool

线程池，管理异步任务执行。

**职责：**
- 任务队列管理
- 工作线程管理
- 优雅退出

**设计模式：** 单例模式

### 4. Router

路由器，管理请求/响应处理。

**职责：**
- 路由注册
- 过滤器链
- 处理器分发

## 协议设计

### 协议头（28 字节）

| 字段 | 大小 | 说明 |
|------|------|------|
| Magic | 4B | 魔数 "TYKE" |
| MsgType | 4B | 消息类型 |
| Reserved | 12B | 保留字段 |
| MetadataLen | 4B | 元数据长度 |
| ContentLen | 4B | 内容长度 |

### 帧格式

```
┌──────────────────────────────────────┐
│ Frame Length (4B)                    │
├──────────────────────────────────────┤
│ Frame Type (1B)                      │
├──────────────────────────────────────┤
│ Payload (Encrypted)                  │
│ ...                                  │
└──────────────────────────────────────┘
```

## 加密机制

### 握手流程

```
Client                          Server
  │                               │
  │──── ECDH Public Key ────────>│
  │                               │
  │<──── ECDH Public Key ────────│
  │                               │
  │     Compute Shared Secret    │
  │                               │
  │<──── Encrypted Data ────────>│
```

### 加密算法

- **密钥交换**：ECDH P-256
- **对称加密**：AES-256-GCM
- **密钥派生**：SHA-256

## 线程模型

### IO 线程

- Windows：IOCP 工作线程
- Linux：epoll 事件循环

### 工作线程

- 线程池处理业务逻辑
- 避免阻塞 IO 线程

## 内存管理

### 对象池

- `TykeRequest` 和 `TykeResponse` 使用对象池
- 减少内存分配开销

### 智能指针

- 使用 `std::shared_ptr` 管理客户端上下文
- 自动释放资源
